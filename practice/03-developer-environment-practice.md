# Track 3 Practice: Developer Environment

This workbook covers practice for:

- Module 10: Structuring a Python Project
- Module 11: Python Packages and Environments

## Module 10 Practice: Structuring a Python Project

### Quick Questions

1. Why does project structure matter more as code grows?
2. Why should tests have a clear home?
3. What should be easy to identify in a good project layout?

### Short Tasks

1. Sketch a simple folder structure for a terminal-based todo app.
2. Decide where you would place:
   - the entry point
   - reusable helper functions
   - tests
   - documentation
3. Explain why your structure makes sense.

## Module 11 Practice: Python Packages and Environments

### Quick Questions

1. What problem does a virtual environment solve?
2. Why is global installation risky?
3. When might Conda be more useful than `pip` plus `venv`?

### Short Tasks

1. Write down the commands you would use to create a virtual environment.
2. Write down the command to install a package with `pip`.
3. Explain why dependency files matter for collaboration.

## Mini Challenge

Design the setup plan for a beginner Python project called `study_tracker`.

Your plan should include:

- a folder structure
- where the main program starts
- where dependencies are recorded
- whether you would use `venv` or Conda
- why your setup is maintainable

## Self-Check

- Can I explain a clean project layout without copying examples?
- Do I understand environment isolation well enough to set up a new project on my own?
