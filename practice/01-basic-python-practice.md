# Track 1 Practice: Basic Python

This workbook covers practice for:

- Module 1: Variables and Data Types
- Module 2: Operators and String Manipulation
- Module 3: Conditionals and Control Flow
- Module 4: Data Structures and Functions
- Module 5: Error Handling

## Module 1 Practice: Variables and Data Types

### Quick Questions

1. What is the difference between a variable name and the value it refers to?
2. Why is `"42"` different from `42`?
3. What does `None` usually represent?

### Short Coding Tasks

1. Create variables for a student's name, age, GPA, and active status.
2. Print the type of each variable using `type()`.
3. Reassign one variable to a different type and explain what changed.

### Reflection

- Which type is easiest to understand for you right now?
- Which type still feels confusing?

## Module 2 Practice: Operators and String Manipulation

### Quick Questions

1. What is the difference between `=` and `==`?
2. What does `%` return?
3. Why are f-strings useful?

### Short Coding Tasks

1. Create two numbers and print their sum, difference, product, and remainder.
2. Create a full name from first name and last name variables.
3. Convert a sentence to uppercase, lowercase, and title case.
4. Build a sentence with an f-string that includes a name and age.

### Reflection

- When does string formatting feel clearer than string concatenation?

## Module 3 Practice: Conditionals and Control Flow

### Quick Questions

1. What is control flow?
2. When would you use `for` instead of `while`?
3. Name three falsy values in Python.

### Short Coding Tasks

1. Write a program that prints whether a number is positive, zero, or negative.
2. Write a grading rule using `if`, `elif`, and `else`.
3. Use a `for` loop to print each character in a word.
4. Use a `while` loop to count from 1 to 5.

### Reflection

- Which kind of branching is easier for you to reason about?

## Module 4 Practice: Data Structures and Functions

### Quick Questions

1. When is a list better than a set?
2. When is a dictionary better than a list?
3. What is the difference between a parameter and a return value?

### Short Coding Tasks

1. Create a list of five favorite topics and print the first and last item.
2. Create a set from repeated values and inspect the result.
3. Create a dictionary describing a book with title, author, and year.
4. Write a function `greet(name)` that returns a greeting.
5. Write a function `average(a, b, c)` that returns the average of three numbers.

### Reflection

- Which data structure feels most natural to you?
- Which one feels most powerful?

## Module 5 Practice: Error Handling

### Quick Questions

1. What is an exception?
2. Why is specific error handling usually better than broad error handling?
3. What is the purpose of `finally`?

### Short Coding Tasks

1. Ask the user for a number and safely convert it using `try` and `except`.
2. Write a function that raises `ValueError` when an input is empty.
3. Write a small example using `try`, `except`, `else`, and `finally`.

### Reflection

- Which kinds of errors do you think are most common for beginners?

## Mini Challenge

Build a simple terminal program that:

- asks for a user's name and age
- validates the age input
- prints a friendly message with f-strings
- stores the result in a dictionary
- uses a function for greeting logic
