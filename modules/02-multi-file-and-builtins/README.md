# Track 2: Multi-File and Built-ins

This track teaches how Python code grows beyond one file.

That shift matters because real software is not written as one long script.
As projects grow, you need:

- separation of responsibilities
- reusable modules
- documentation habits
- file handling
- stronger familiarity with Python's built-in tools

## Lessons in This Track

1. [Modules and `__main__`](./01-modules-and-main.md)
2. [Docstrings](./02-docstrings.md)
3. [Python Standard Library](./03-python-standard-library.md)
4. [File I/O](./04-file-io.md)

## What You Should Understand After This Track

- how Python imports work
- why code is split across files
- how docstrings improve maintainability
- how the standard library saves time
- how programs read and write files safely
